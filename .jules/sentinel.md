## 2024-06-04 - Missing Authorization on CI Workflows
**Vulnerability:** GitHub Actions workflow executing Claude Code on issue and PR comments did not check the `author_association` of the user triggering the event. Any user (including malicious outsiders) could trigger the action simply by mentioning `@claude` in an issue or PR comment.
**Learning:** Workflows triggered by `issue_comment`, `pull_request_review_comment`, `pull_request_review`, and `issues` are executed in the context of the base repository. If they do not verify authorization, they can be abused by untrusted actors, potentially exposing secrets (like `CLAUDE_CODE_OAUTH_TOKEN` and standard `GITHUB_TOKEN`) to excessive execution or exploitation if the workflow takes inputs.
**Prevention:** Always verify `author_association` in the `if` condition of GitHub Actions workflows triggered by public inputs (like comments). Allow only `OWNER`, `MEMBER`, or `COLLABORATOR`.
## 2024-10-27 - Path Traversal in FastAPI SPA Fallback
**Vulnerability:** The SPA fallback route in `server/pivot/api/app.py` directly appended the user-provided path to the `dist` directory path using `dist / path` without verifying that the resolved target was still inside the `dist` directory. This allowed a path traversal attack where an attacker could access sensitive files outside the `dist` directory by using `../` in the path.
**Learning:** Custom static file serving or fallback routes must always resolve paths and check if they are relative to the intended base directory. `Path.resolve()` followed by `target.is_relative_to(base.resolve())` is essential for validating paths.
**Prevention:** Always validate that user-provided paths (even in wildcard routes) cannot escape their sandbox. Use established libraries for static files when possible, or rigorously sanitize and validate resolved paths for custom fallback handlers.
